# Описание проекта Polytech-Universe

Этот каталог содержит итоговую рабочую версию проекта по анализу данных спутника Polytech-Universe. Документ служит единственной справкой по репозиторию: здесь кратко описаны назначение файлов, логика обработки данных, основные результаты и то, какие файлы удобно открыть при проверке работы.

## 1. Краткая навигация

- Если нужно посмотреть код анализа исходных данных: `Скрипты/01_анализ_исходных_данных/`.
- Если нужно посмотреть код учебного набора: `Скрипты/02_учебный_набор/`.
- Если нужно положить или проверить исходные файлы задания: `Исходные данные/`.
- Если нужны готовые графики и таблицы по исходным данным: `Задание 1 · Анализ данных/`.
- Если нужен учебный полусинтетический набор и его проверочные графики: `Задание 2 · Данные для обучения/`.
- Если нужен самый короткий текстовый итог по исходным наборам: `Задание 1 · Анализ данных/summaries/analysis_summary_200_stab.txt` и `Задание 1 · Анализ данных/summaries/analysis_summary_201_ori.txt`.

## 2. Основная идея проекта

Работа состоит из двух связанных частей. В первой части анализируются реальные исходные данные: измерения магнитометра и навигационные точки спутника. Скрипты очищают данные, сопоставляют магнитные измерения с координатами, считают статистики и строят графики.

Во второй части на основе реальных данных создается учебный полусинтетический набор. В нем используются реальные навигационные фрагменты, расчет теоретического магнитного поля по IGRF-14, остаточная ошибка реальных измерений и специально добавленные учебные аномалии. 

## 3. Описание файлов и папок

### Скрипты MATLAB

- `Скрипты/01_анализ_исходных_данных/analyze_200_stab.m` - основной анализ набора `200_stab`: очистка магнитометра, чтение навигации, синхронизация, статистика, графики и текстовая сводка.
- `Скрипты/01_анализ_исходных_данных/analyze_201_ori.m` - анализ набора `201_ori`; дополнительно формирует отфильтрованную версию `201_ori_filtered` для сильных выбросов магнитного поля.
- `Скрипты/01_анализ_исходных_данных/plot_ground_track_magnetic_norm.m` - строит наземный трек, где цвет точки показывает модуль магнитного поля `|B|`.
- `Скрипты/01_анализ_исходных_данных/plotUnifiedGroundTrackMap.m` - вспомогательная функция для единого оформления карт наземного трека.
- `Скрипты/02_учебный_набор/generate_teaching_dataset.m` - генерирует учебный полусинтетический набор данных, raw-like файлы, метаданные и контрольные графики.
- `Скрипты/02_учебный_набор/analyze_teaching_dataset.m` - повторно анализирует учебный набор и строит диагностические графики по сегментам, магнитному полю и аномалиям.
- `Скрипты/02_учебный_набор/defaultTeachingGeneratorConfig.m` - конфигурация генератора учебного набора: даты сегментов, параметры остатка, список учебных аномалий и настройки генерации.

### Исходные данные

Папка `Исходные данные/` предназначена для локального размещения исходных файлов задания:

```text
200_magn_stab.txt
200_lla_stab.csv
201_magn_ori.txt
201_lla_ori.csv
```

Файлы `*_magn_*.txt` содержат измерения магнитометра: время и компоненты магнитного поля. Файлы `*_lla_*.csv` содержат навигационные данные: время, высоту, широту и долготу. Эти файлы считаются входными данными и обычно не добавляются в Git.

## 4. Как получались данные

Для исходных наборов `200_stab` и `201_ori` обработка идет так:

1. Читаются файлы магнитометра и навигации.
2. Из магнитометра удаляются нечисловые строки, полностью нулевые строки и некорректные измерения.
3. Для части строк с поврежденным временем выполняется восстановление временной метки, если это можно сделать по соседним измерениям.
4. Рассчитывается модуль магнитного поля `|B| = sqrt(Bx^2 + By^2 + Bz^2)`.
5. Каждая навигационная точка сопоставляется с ближайшим по времени магнитным измерением.
6. Считаются статистики качества данных, синхронизации, скачков и орбитального периода.
7. Строятся графики магнитного поля и наземного трека.

Для набора `201_ori` дополнительно создана фильтрованная версия `201_ori_filtered`. Она нужна, чтобы отдельно показать поведение данных после удаления одиночных экстремальных выбросов `|B|`.

Учебный набор создается по другой логике:

1. Берутся реальные навигационные точки и реальные остатки магнитного поля.
2. Для координат рассчитывается теоретическое поле **по IGRF-14.**
3. Реалистичный остаток добавляется к модельному полю.
4. Один и тот же исходный фрагмент переносится в несколько учебных временных сегментов.
5. В данные добавляются контролируемые ошибки: скачки координат, скачки высоты, магнитные пики, провалы, ступеньки, пропуски и поврежденные временные метки.
6. Набор сохраняется в обычной таблице и в raw-like формате, чтобы его можно было анализировать теми же методами, что и исходные файлы.

## 5. Краткий отчет по результатам

Набор `200_stab` используется как более стабильный пример исходных данных. По нему построены основные статистики магнитного поля, наземный трек и сводка качества синхронизации. Он удобен как базовый набор для объяснения полного процесса обработки.

Набор `201_ori` содержит более заметные экстремальные значения магнитного поля. Поэтому для него сохранены две версии результатов: исходная и `201_ori_filtered`. Сравнение этих файлов показывает, как одиночные выбросы влияют на статистику и визуализацию.

Графики наземного трека показывают, где находился спутник во время измерений. Если цвет задан высотой, удобно проверять непрерывность навигации; если цвет задан `|B|`, можно визуально искать участки с необычным магнитным поведением.

Учебный набор нужен не как научный продукт, а как демонстрационный материал. Его ценность в том, что аномалии заранее размечены, поэтому по нему можно проверять алгоритмы поиска ошибок и объяснять преподавателю, как разные типы проблем выглядят в таблицах и на графиках.
